# AI가 찾은 스타 애널리스트 어워즈 시스템 개발 기획안

## 1. 프로젝트 개요

### 1.1 프로젝트명
**AI가 찾은 스타 애널리스트** (가제)

### 1.2 목적
인공지능(AI)이 직접 빅데이터를 분석해 대한민국 최고의 애널리스트를 선정하는 새로운 개념의 어워즈 시스템 구축. 증권사 현직 애널리스트의 과거 애널리스트 보고서를 기반으로 실제 실적과 예측의 정확도를 평가하고, AI가 인터넷상의 실적 데이터 등을 수집하여 사실에 입각한 분석과 scoring을 수행하여 award를 수여하기 위한 시스템

### 1.3 추진 배경
- '금융포럼'의 부재를 대체할 수익모델 및 신사업 구축
- 증권업계에 이코노미스트의 브랜드 확장
- 조선일보 및 경제지 등에서 실시하는 기존 방식의 문제점을 AI로 해결

### 1.4 핵심 목표
- 소수 전문가의 정성 평가에 의존하는 기존 방식의 한계를 AI 기반의 새로운 패러다임으로 전환
- 리포트의 실제 시장 영향력 측정 한계를 리포트 성과(정확도)와 시장 영향력(파급력) 동시 측정으로 해결
- 데이터로 '진짜 영향력' 있는 애널리스트 발굴
- AI 기반 자동화된 평가 프로세스 구축
- 투명하고 공정한 award 선정 시스템

## 2. 시스템 아키텍처

### 2.1 전체 구조
```
┌─────────────────────────────────────────────────────────────┐
│                    웹 애플리케이션 (Frontend)                  │
│  - 애널리스트 보고서 업로드                                    │
│  - 평가 결과 조회 및 시각화                                    │
│  - Award 결과 확인                                            │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────┴──────────────────────────────────┐
│              API 서버 (Backend)                              │
│  - 보고서 파싱 및 데이터 추출                                 │
│  - 평가 프로세스 오케스트레이션                               │
│  - 결과 저장 및 조회                                         │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────┴──────────────────────────────────┐
│         AI Agent Orchestration Layer                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ OpenAI   │  │ Claude   │  │ Gemini   │  │Perplexity│   │
│  │ Agent    │  │ Agent    │  │ Agent    │  │ Agent    │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│                                                              │
│         Mix of Agents 추론 알고리즘                          │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 기술 스택
- **Frontend**: React / Next.js (대시보드 및 시각화)
- **Backend**: Python FastAPI / Django (API 서버)
- **Database**: PostgreSQL (보고서 메타데이터, 평가 결과)
- **Vector DB**: Pinecone / Weaviate (보고서 임베딩 저장)
- **AI APIs**: OpenAI, Anthropic Claude, Google Gemini, Perplexity
- **데이터 수집**: 웹 스크래핑, 금융 데이터 API (예: 한국거래소, 금융감독원)

## 3. 핵심 기능 요건

### 3.1 보고서 수집 및 파싱
- **입력**: PDF/Word 형식의 애널리스트 보고서
- **추출 항목**:
  - 보고서 메타데이터 (작성일, 애널리스트명, 증권사명, 대상 기업)
  - 예측 항목:
    - 매출액 예측
    - 영업이익 예측
    - 순이익 예측
    - 주가 목표가 예측
    - 투자의견 (Buy/Hold/Sell)
    - 목표 기간 (3개월, 6개월, 1년 등)
  - 예측 근거 및 논리 구조

### 3.2 실적 데이터 수집 및 검증
- **외부 데이터 소스**:
  - 한국거래소 공시 시스템 (DART)
  - 금융감독원 전자공시시스템
  - 각종 뉴스 및 산업 리포트
  - 주가 데이터 API
- **수집 항목**:
  - 실제 실적 데이터 (매출액, 영업이익, 순이익)
  - 실제 주가 변동
  - 예측 시점 이후의 재무제표
  - 산업 동향 및 시장 환경 변화

### 3.3 AI 기반 평가 및 Scoring

#### 3.3.1 Mix of Agents 아키텍처
각 AI 에이전트가 특화된 역할을 수행:

**1. OpenAI Agent (GPT-4)**
- 역할: 보고서 구조 분석 및 핵심 예측 추출
- 입력: 원본 보고서 텍스트
- 출력: 구조화된 예측 데이터 (JSON)

**2. Claude Agent (Claude 3.5 Sonnet)**
- 역할: 예측 근거 분석 및 논리 검증
- 입력: 추출된 예측 + 근거 문단
- 출력: 예측 신뢰도 점수, 논리 일관성 평가

**3. Gemini Agent (Gemini Pro)**
- 역할: 실적 데이터 검증 및 비교 분석
- 입력: 예측값 + 실제값
- 출력: 정확도 점수, 오차율 계산, 트렌드 분석

**4. Perplexity Agent**
- 역할: 외부 데이터 수집 및 사실 검증
- 입력: 기업명, 예측 시점, 예측 항목
- 출력: 관련 실적 데이터, 뉴스, 공시 정보

#### 3.3.2 추론 알고리즘 (Mix of Agents)
```
1단계: 보고서 파싱 (OpenAI)
   └─> 구조화된 예측 데이터 추출

2단계: 데이터 수집 (Perplexity)
   └─> 실제 실적 데이터 검색 및 수집

3단계: 근거 분석 (Claude)
   └─> 예측 근거의 논리성 및 신뢰도 평가

4단계: 정확도 계산 (Gemini)
   └─> 예측 vs 실제 비교 분석

5단계: 통합 Scoring (Ensemble)
   └─> 각 Agent 결과를 가중 평균하여 최종 점수 산출
```

### 3.4 3단계 심사 프로세스

#### 3.4.1 1단계: AI 정량 분석
**목적**: AI 기반 정량적 데이터 분석을 통한 객관적 평가

**평가 항목**:
- **리포트 생산량 및 발간 주기**
  - 보고서 발간 빈도
  - 보고서 생산량 (월별, 분기별)
  - 보고서 일관성 (정기 발간 여부)

- **종목 리딩 정확도 (목표주가 대비 수익률)**
  - 목표주가 예측 대비 실제 주가 수익률
  - 투자의견 적중률 (Buy/Hold/Sell)
  - 예측 기간 대비 실제 수익률

- **산업별 분석 깊이 및 통찰력**
  - 산업별 전문성 깊이
  - 분석 통찰력 평가
  - 산업 트렌드 파악 정확도

**AI Agent 활용**:
- OpenAI Agent: 보고서 구조 분석 및 핵심 예측 추출
- Perplexity Agent: 실제 실적 데이터 수집 및 검증
- Claude Agent: 예측 근거 분석 및 논리 검증
- Gemini Agent: 실적 데이터 검증 및 비교 분석

#### 3.4.2 2단계: SNS·시장 반응 분석
**목적**: 리포트의 실제 시장 영향력 및 파급력 측정

**평가 항목**:
- **X·블로그 등의 SNS 언급량**
  - SNS 플랫폼별 언급 횟수
  - 긍정/부정 감성 분석
  - 영향력 있는 계정의 언급 여부

- **관련 콘텐츠 바이럴 지수**
  - 콘텐츠 확산 속도
  - 공유 및 재게시 횟수
  - 바이럴 지수 계산

- **투자자 커뮤니티 반응 지수**
  - 투자자 커뮤니티(네이버 카페, 페이스북 그룹 등) 반응
  - 토론 및 논의 활성도
  - 커뮤니티 영향력 점수

**데이터 소스**:
- SNS API (X(Twitter), 블로그, 인스타그램)
- 투자자 커뮤니티 크롤링
- 뉴스 미디어 언급량

#### 3.4.3 3단계: 전문가 평가 및 설문
**목적**: 전문가 및 실제 투자자 의견을 반영한 정성 평가

**평가 항목**:
- **운용사 및 리서치센터 대상 설문**
  - 운용사 애널리스트 평가
  - 리서치센터 평가
  - 전문가 의견 수렴

- **리테일 투자자 대상 설문**
  - 일반 투자자 설문
  - 투자 의사결정에 미친 영향도
  - 리포트 유용성 평가

**평가 방식**:
- 온라인 설문 조사
- 전문가 패널 평가
- 투자자 투표

#### 3.4.4 최종 Scoring 체계

**가중치 설정**:
1. **AI 정량 분석 점수 (40%)**
   - 리포트 생산량: 20%
   - 종목 리딩 정확도: 50%
   - 산업별 분석 깊이: 30%

2. **SNS·시장 반응 점수 (30%)**
   - SNS 언급량: 40%
   - 바이럴 지수: 30%
   - 투자자 커뮤니티 반응: 30%

3. **전문가 평가 및 설문 점수 (30%)**
   - 운용사/리서치센터 설문: 60%
   - 리테일 투자자 설문: 40%

**Scoring 공식**:
```
최종 점수 = (AI 정량 분석 점수 × 0.4) 
          + (SNS·시장 반응 점수 × 0.3) 
          + (전문가 평가 및 설문 점수 × 0.3)

각 점수는 0-100점 척도로 정규화
```

### 3.5 심사 대상

#### 3.5.1 평가 대상
- **일부 섹터 애널리스트 평가**: AI, 2차전지, 방산, IPO 등 최근 한국 경제 핵심 성장 동력으로 주목받는 주요 섹터의 애널리스트
- **섹터별 전문성 평가**: 각 섹터별 전문 애널리스트의 깊이 있는 분석 평가

#### 3.5.2 평가 기간
- 연간 평가를 기본으로 하되, 분기별 중간 평가도 가능
- 최소 1년 이상의 보고서 이력을 가진 애널리스트 대상

### 3.6 Award 선정 시스템
- **기간별 평가**: 월별, 분기별, 연간
- **카테고리별 수상**: 
  - 섹터별 (AI, 2차전지, 방산, IPO 등)
  - 산업별
  - 애널리스트 경력별 (신인, 중견, 시니어)
- **랭킹 시스템 및 시각화**: 각 카테고리별 랭킹 및 상세 분석 결과 제공

- 데이터베이스 설계 및 구축
- AI API 연동 (4개 모두)
- 보고서 파싱 모듈 개발

### Phase 2: AI Agent 개발 (6주)
- 각 AI 에이전트 개별 개발

- 대시보드 UI/UX
- 평가 결과 시각화

### Phase 6: 테스트 및 최적화 (4주)
- 통합 테스트
